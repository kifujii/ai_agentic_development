# 悪いプロンプト例集

## Terraformコード生成

### 例1: 曖昧なプロンプト
```
EC2を作成して
```

### 例2: 不足パラメータ
```
VPCとEC2を作成してください
```

### 例3: 要件が不明確
```
S3バケットを作成
```

### 例4: 既存リソースとの衝突を考慮していない
```
セキュリティグループを作成してください
```

## Ansible Playbook生成

### 例1: 曖昧なプロンプト
```
サーバーを再起動するPlaybookを作成してください
```

### 例2: 不足パラメータ
```
パッケージをインストールしてください
```

### 例3: 冪等性を考慮していない
```
サービスを開始してください
```

## 共通の問題点

1. **不足パラメータ**: リージョン、インスタンスタイプ、AMIなど必要な情報が不足
2. **不明確な要件**: 「適切に」「適切な」などの曖昧な表現
3. **既存リソースとの衝突**: 既存のリソース情報を考慮していない
4. **ベストプラクティスの欠如**: セキュリティ、コスト、パフォーマンスを考慮していない
5. **エラーハンドリングの欠如**: エラー発生時の処理が不明確
