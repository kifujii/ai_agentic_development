# プロンプト改善ガイド

## プロンプト改善のステップ

### ステップ1: 不足パラメータの特定

悪いプロンプトで生成されたコードを確認し、不足しているパラメータを特定します。

**例**:
- 悪いプロンプト: "EC2を作成して"
- 不足パラメータ: リージョン、インスタンスタイプ、AMI、セキュリティグループ

### ステップ2: 明確な要件定義の追加

不足パラメータを明記し、明確な要件定義を追加します。

**例**:
```
ap-northeast-1リージョンに、t3.microインスタンスタイプのEC2インスタンスを作成してください。
AMIはAmazon Linux 2023を使用し、セキュリティグループはSSH（ポート22）のみ許可してください。
```

### ステップ3: 不足パラメータの聞き返し指示を追加

「足りていないパラメータがある場合は聞き返してください」を追加します。

**例**:
```
注意事項:
- 足りていないパラメータなどがある場合は、そのまま構築するのではなく一度聞き返してください
```

### ステップ4: 既存リソースとの衝突回避指示を追加

既存リソースとの衝突を回避する指示を追加します。

**例**:
```
注意事項:
- 既存のVPCやサブネットと衝突しないように確認してください
```

### ステップ5: ベストプラクティスの指示を追加

ベストプラクティスに従う指示を追加します。

**例**:
```
注意事項:
- ベストプラクティスに従ってください
- 変数定義を含めてください
- コメントを適切に追加してください
```

## プロンプト改善のチェックリスト

- [ ] 必要なパラメータがすべて明記されているか
- [ ] 要件が明確で曖昧な表現がないか
- [ ] 「足りていないパラメータがある場合は聞き返してください」が含まれているか
- [ ] 既存リソースとの衝突回避指示が含まれているか
- [ ] ベストプラクティスの指示が含まれているか
- [ ] 出力形式が明確に指定されているか

## 改善前後の比較

### 改善前（悪いプロンプト）
```
EC2を作成して
```

### 改善後（良いプロンプト）
```
下記条件を満たすEC2インスタンスを構築するTerraformコードを生成してください。

要件:
- リージョン: ap-northeast-1
- インスタンスタイプ: t3.micro
- OS: Amazon Linux 2023
- セキュリティグループ: SSH（ポート22）のみ許可、送信は全許可
- タグ: Name = "training-ec2", Environment = "training"

注意事項:
- 足りていないパラメータなどがある場合は、そのまま構築するのではなく一度聞き返してください
- 既存のEC2インスタンスと衝突しないように確認してください
- 変数定義を含めてください
- コメントを適切に追加してください
- ベストプラクティスに従ってください
```

## フィードバックループ

1. **生成コードの確認**: 生成されたコードを確認
2. **不足点の指摘**: 不足しているパラメータや問題点を特定
3. **プロンプト改善**: 不足点を反映してプロンプトを改善
4. **再生成**: 改善したプロンプトでコードを再生成
5. **品質確認**: 生成コードの品質を確認し、必要に応じて再度改善
