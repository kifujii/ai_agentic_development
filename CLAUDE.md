# AI駆動IaCワークショップ

このプロジェクトは、Terraform / Ansible を使ったインフラ構築を AI Agent と協力して学ぶ研修ワークショップです。

## ルール

- 受講者から与えられた指示に基づいて作業してください
- **先回りして次のステップの作業を行わないでください**
- 受講者から指示された **1つのタスクだけ** を実行してください
- タスクが完了したら結果を報告し、**次の指示を待ってください**
- 受講者が明示的に依頼していない作業は行わないでください
- コードを生成する際は、受講者が理解できるよう各部分の意味を簡潔に説明してください

## アクセス制限（重要）

このプロジェクトは**学習用ハンズオンワークショップ**です。受講者が自分で考えて学ぶことが目的です。
以下のファイル・ディレクトリは**セッションの解答や評価基準が含まれている**ため、**絶対に読まないでください**：

- `docs/session_guides/` — セッションガイド（プロンプト例・完成形コードが記載されている）
- `docs/TRAINING_MENU.md` — トレーニングメニュー（各セッションの目標・手順が記載されている）
- `docs/images/` — アーキテクチャ構成図（目標構成が記載されている）
- `evaluation/` — 評価チェックリスト（採点基準が記載されている）
- `scripts/check.sh` — 完了チェックスクリプト（期待される成果物の定義が含まれている）
- `scripts/break_session.sh` — 障害シミュレーションスクリプト（障害パターンが記載されている）

**受講者が口頭またはプロンプトで伝えた内容のみに基づいて作業してください。**
上記のファイルを参照して先回りすることは、受講者の学習効果を著しく損ないます。

## リソース命名規則（重要）

複数の受講者が同一のAWS環境を使用しています。**すべてのAWSリソース名には必ず prefix を付けてください。**

- Terraform の場合: `var.prefix` 変数を使用（環境変数 `TF_VAR_prefix` から自動取得）
- AWS CLI の場合: 環境変数 `$PREFIX` を使用
- 例: VPC名 → `${var.prefix}-vpc`、IAMロール名 → `${PREFIX}-ec2-agent-role`

**prefix を付けずにリソースを作成すると、他の受講者のリソースと名前が衝突します。**

## プロジェクト構成

- `terraform/` — Terraform の設定ファイル（受講者が作成）
- `ansible/` — Ansible の設定・Playbook（受講者が作成）
- `keys/` — SSH鍵（Git管理外）
- `scripts/` — チェック・障害シミュレーション用スクリプト

## 接続情報

- SSH鍵: `keys/training-key`（プロジェクトルートからの相対パス）
- SSH鍵（絶対パス）: `~/workspace/keys/training-key`（Ansible の設定ファイルではこちらを使用）
- SSHユーザー: `ec2-user`
- EC2のIPアドレスは受講者から指示されます

## 環境

- ハンズオン環境: ブラウザ版 VSCode（code-server on AWS EC2）
- ツール（Terraform, Ansible, AWS CLI, Claude Code）はプリインストール済み
- AWS認証情報は環境に事前設定済み
