# セッション3：システム運用基礎とAnsible Playbook作成 評価チェックリスト

## 評価項目

### 1. Ansibleインベントリの設定（20点）

#### 1.1 インベントリファイルの作成（10点）
- [ ] インベントリファイル（inventory.ini）を作成した
- [ ] EC2インスタンスの情報を正しく設定した
- [ ] SSH接続設定を正しく記述した

#### 1.2 SSH接続設定（10点）
- [ ] SSH鍵ファイルのパスを設定した
- [ ] キーペアファイルの権限を設定した（chmod 400）
- [ ] StrictHostKeyCheckingの設定を追加した

### 2. 接続テスト（15点）

#### 2.1 Ansible接続テスト（10点）
- [ ] `ansible all -i inventory.ini -m ping`が成功した
- [ ] すべてのホストに接続できた

#### 2.2 システム情報の取得（5点）
- [ ] `ansible all -i inventory.ini -m setup`でシステム情報を取得した
- [ ] 情報が正しく取得できた

### 3. サーバー再起動の自動化（35点）

#### 3.1 Playbookの作成（20点）
- [ ] restart_server.ymlを作成した
- [ ] タスクを適切に定義した
- [ ] ハンドラーを適切に使用した
- [ ] エラーハンドリングを含めた

#### 3.2 再起動前後の状態確認（10点）
- [ ] 再起動前の状態を確認するタスクを追加した
- [ ] 再起動後の状態を確認するタスクを追加した
- [ ] 状態を表示する機能を実装した

#### 3.3 Playbookの実行（5点）
- [ ] `ansible-playbook`でPlaybookを実行した
- [ ] 実行が成功した

### 4. その他の基本タスク（30点）

## 評価基準

### 優秀（90点以上）
- すべての項目を完了し、追加のタスクを実装した
- Playbookがベストプラクティスに従っている
- 適切なエラーハンドリングとロールバック機能がある

### 良好（70-89点）
- 必須項目をすべて完了した
- Playbookが正常に動作する
- 基本的なタスクが実装されている

### 要改善（70点未満）
- 一部の項目が未完了
- Playbookが正常に動作しない
- エラーハンドリングが不十分
