# セッション4：Ansible Playbook生成・実行自動化エージェント開発 評価チェックリスト

## 評価項目

### 1. Ansible Playbook生成エージェントの実装（35点）

#### 1.1 Prompt Engineering（10点）
- [ ] Ansible Playbook生成用のプロンプトを設計した
- [ ] Ansible特有の構文を理解している
- [ ] タスク構造を最適化するプロンプトを作成した

#### 1.2 自然言語からPlaybookへの変換（15点）
- [ ] 自然言語指示をPlaybookに変換する機能を実装した
- [ ] タスクの分解と構造化を実装した
- [ ] 適切なモジュールを選択できる

#### 1.3 タスク構造の最適化（10点）
- [ ] 冪等性を確保する機能を実装した
- [ ] エラーハンドリングを組み込む機能を実装した
- [ ] ハンドラーの最適化を実装した

### 2. Ansible実行自動化の実装（30点）

#### 2.1 ansible-playbookの自動実行（15点）
- [ ] ansible-playbookの自動実行機能を実装した
- [ ] 一時ファイルの管理を実装した
- [ ] 実行結果を取得する機能を実装した

#### 2.2 実行結果の検証とレポート生成（10点）
- [ ] 実行ログを解析する機能を実装した
- [ ] 成功/失敗を判定する機能を実装した
- [ ] レポートを生成する機能を実装した

#### 2.3 エラーハンドリングとリトライ機能（5点）
- [ ] 失敗時の処理を実装した
- [ ] リトライ機能を実装した（推奨）

### 3. エージェントの動作確認とテスト（35点）

#### 3.1 基本的な動作確認（15点）
- [ ] 「監視エージェントをインストール」などの指示でPlaybookを生成できる
- [ ] 生成されたPlaybookが正しい形式である
- [ ] Playbookが正常に実行できる

#### 3.2 複数のタスクでの動作確認（15点）
- [ ] 「サーバ情報を取得」などのタスクで動作する
- [ ] 複数のタスクを処理できる
- [ ] 生成されたPlaybookの品質が高い

#### 3.3 エラーハンドリングの確認（5点）
- [ ] 無効な指示に対して適切なエラーを返す
- [ ] 実行エラー時に適切に処理する

## 評価基準

### 優秀（90点以上）
- すべての項目を完了し、追加の機能を実装した
- エージェントが安定して動作し、エラーハンドリングが適切
- 生成されたPlaybookの品質が高い
- 複雑なタスクにも対応できる

### 良好（70-89点）
- 必須項目をすべて完了した
- エージェントが基本的に動作する
- シンプルなタスクに対応できる

### 要改善（70点未満）
- 一部の項目が未完了
- エージェントが正常に動作しない
- 生成されたPlaybookにエラーがある
