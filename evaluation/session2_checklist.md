# セッション2：VPC/EC2の設計・構築・検証 評価チェックリスト

## 評価項目

### 1. 目標構成の理解（10点）

- [ ] Mermaid図で提示された目標構成を理解した
- [ ] VPC、サブネット、EC2の関係を理解した
- [ ] セキュリティグループの設計を理解した

### 2. Agent形式での構築（35点）

#### 2.1 VPC/サブネットの構築（15点）
- [ ] VPCを作成した（CIDR: 10.0.0.0/16）
- [ ] パブリックサブネットを作成した（10.0.1.0/24）
- [ ] インターネットゲートウェイを作成した
- [ ] ルートテーブルを適切に設定した

#### 2.2 セキュリティグループの設計（10点）
- [ ] SSH（ポート22）のインバウンドルールを設定した
- [ ] アウトバウンドルールを設定した

#### 2.3 EC2インスタンスの構築（10点）
- [ ] EC2インスタンスを作成した（t3.micro, Amazon Linux 2023）
- [ ] SSH接続用のキーペアを設定した
- [ ] パブリックサブネットに配置した

### 3. Agent開発の活用（20点）

- [ ] Prompt Engineeringを実践した（要件の明確な伝え方）
- [ ] Context Engineeringを実践した（既存AWSリソース情報の活用）
- [ ] 承認ワークフローを体験した
- [ ] エラー修正プロセスを体験した
- [ ] 反復的改善プロセスを体験した

### 4. 構築結果の検証（25点）

#### 4.1 Terraform実行確認（10点）
- [ ] `terraform plan` で実行計画を確認した
- [ ] `terraform apply` でリソースを作成した
- [ ] `terraform output` で構築結果を確認した

#### 4.2 SSH接続テスト（10点）
- [ ] キーペアファイルの権限を設定した（chmod 400）
- [ ] SSH接続テストに成功した

#### 4.3 リソース確認（5点）
- [ ] `terraform state list` でリソース一覧を確認した

### 5. 振り返り（10点）

- [ ] Prompt Engineeringの効果を振り返った
- [ ] Context Engineeringの重要性を振り返った
- [ ] フィードバックループの体験を振り返った
- [ ] 構築結果の検証方法を振り返った
