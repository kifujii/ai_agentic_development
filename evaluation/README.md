# 評価チェックリスト

各セッションの成果物評価チェックリストです。

## チェックリスト一覧

- [セッション0](session0_checklist.md) - Claude Code に慣れよう
- [セッション1](session1_checklist.md) - VPC + EC2 を段階的に構築
- [セッション2](session2_checklist.md) - Webアプリケーションを公開
- [セッション3](session3_checklist.md) - HTTPS 対応（任意）
- [セッション4](session4_checklist.md) - サーバー再起動の自動化（Ansible）
- [セッション5](session5_checklist.md) - SSM Agent & CloudWatch Agent 導入
- [セッション6](session6_checklist.md) - サーバー情報取得・運用レポート作成（任意）

## 評価の観点

1. **構築の完成度**: 各ステップが正しく完了しているか
2. **Agent開発の活用**: Agent機能を使って進められたか
3. **理解度**: 各技術要素の目的・仕組みを自分の言葉で説明できるか
4. **振り返り**: プロンプトの工夫点やエラー対処を記録できたか

## チェックリストの構成

各チェックリストは以下の3部構成です：

| セクション | 内容 |
|---|---|
| **ステップ別チェック** | 各ステップの完了確認（操作の完了） |
| **理解度チェック** | 技術的な理解を自分の言葉で記述する問い |
| **プロンプトの振り返り** | Agent開発で工夫した点・学んだ点の記録 |

## 自動チェック

コマンドラインでセッションの完了状態を確認できます：

```bash
# セッション単位でチェック
./scripts/check.sh session1

# 特定のステップだけチェック
./scripts/check.sh session1 step3
```
