# セッション1：AI x IaC基礎実践とAgent開発の理解 評価チェックリスト

## 評価項目

### 1. 環境セットアップの確認（10点）

- [ ] DevSpacesワークスペースにアクセスできた
- [ ] セットアップスクリプトが実行済みである
- [ ] AWS認証情報が正しく設定されている（`aws sts get-caller-identity` が成功）
- [ ] Continueが正常に動作する（Agentモードでファイル作成テストが成功）

### 2. Prompt Engineering実践（30点）

#### 2.1 悪いプロンプトでの体験（10点）
- [ ] 悪いプロンプトでTerraformコードを生成した
- [ ] 生成されたコードの問題点を振り返った（不足パラメータ、不適切なデフォルト値など）
- [ ] `terraform/bad_prompt/` にコードを保存した

#### 2.2 フィードバックループでのプロンプト改善（10点）
- [ ] 悪いプロンプトの問題点を特定した
- [ ] プロンプトを段階的に改善した
- [ ] 改善したプロンプトで再生成し、`terraform/improved_prompt/` に保存した

#### 2.3 振り返り（10点）
- [ ] 改善前後のコード品質の違いを振り返った
- [ ] フィードバックループを通じたプロンプト改善の効果を理解した

### 3. Context Engineering実践（20点）

#### 3.1 コンテキストなしでの生成（10点）
- [ ] コンテキストなしでEC2コードを生成した
- [ ] `terraform/no_context/` にコードを保存した
- [ ] 固定値やデフォルト値による問題点を振り返った

#### 3.2 コンテキストありでの生成（10点）
- [ ] Continueのチャットで環境情報を取得した
- [ ] 取得した情報をコンテキストとしてプロンプトに含めた
- [ ] `terraform/with_context/` にコードを保存した
- [ ] コンテキストの有無による違いを振り返った

### 4. Agent形式での開発の理解（25点）

#### 4.1 Agent形式での実践（15点）
- [ ] Agent形式を選択してContinueを使用した
- [ ] 自然言語で指示し、Agentが実行計画を提示した
- [ ] 承認ワークフローを体験した（計画を確認→承認）
- [ ] `terraform/agent_practice/` にコードが自動生成された
- [ ] `terraform apply` で実際にAWSリソースを構築した
- [ ] 実践後に `terraform destroy` でリソースを削除した

#### 4.2 フィードバックループの体験（10点）
- [ ] 承認ワークフロー（計画提示→承認→実行）を体験した
- [ ] エラー修正プロセス（エラー検出→修正提案→承認）を体験した（エラー発生時）
- [ ] 反復的改善（フィードバック→改善→再検証）を体験した

### 5. チャット形式 vs Agent形式の比較（15点）

- [ ] チャット形式で作成したファイルを確認・振り返りした
- [ ] Agent形式で作成したファイルを確認・振り返りした
- [ ] 両形式の作業時間・手動操作回数・エラー修正効率を比較した
- [ ] それぞれの形式の特徴と適した場面を考察した

## 成果物チェックリスト

### 必須成果物
- [ ] `terraform/bad_prompt/` - 悪いプロンプトで生成されたコード
- [ ] `terraform/improved_prompt/` - 改善したプロンプトで生成されたコード
