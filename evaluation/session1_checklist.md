# セッション1：AI x IaC基礎実践 評価チェックリスト

## 評価項目

### 1. 環境セットアップ（20点）

#### 1.1 OpenShift DevSpaces環境の確認（5点）
- [ ] ワークスペースに正常にアクセスできた
- [ ] 基本的なコマンドが実行できた
- [ ] ファイルシステムへのアクセスが可能だった

#### 1.2 AWS認証情報の設定（5点）
- [ ] AWS CLIがインストールされている
- [ ] `aws configure`で認証情報を設定した
- [ ] `aws sts get-caller-identity`で認証が確認できた
- [ ] 適切なIAM権限があることを確認した

#### 1.3 生成AI APIキーの設定（5点）
- [ ] 環境変数にAPIキーを設定した
- [ ] .envファイルを作成した（推奨）
- [ ] APIキーが有効であることを確認した

#### 1.4 必要なツールのインストール（5点）
- [ ] Terraformがインストールされている
- [ ] Ansibleがインストールされている
- [ ] Pythonパッケージがインストールされている
- [ ] すべてのツールが正常に動作する

### 2. Prompt Engineering実践（30点）

#### 2.1 基本的なプロンプトの作成（10点）
- [ ] 明確な要件定義を含むプロンプトを作成した
- [ ] リソース仕様を具体的に記述した
- [ ] プロンプトからTerraformコードが生成できた

#### 2.2 プロンプトテンプレートの作成（10点）
- [ ] 再利用可能なプロンプトテンプレートを作成した
- [ ] テンプレートに変数を含めた
- [ ] テンプレートを使用してコードを生成した

#### 2.3 プロンプト最適化（10点）
- [ ] 段階的なプロンプト最適化を実践した
- [ ] フィードバックループを構築した
- [ ] 生成コードの品質が向上した

### 3. Context Engineering実践（20点）

#### 3.1 AWSリソース情報のコンテキスト化（10点）
- [ ] AWSリソース情報を取得するコードを作成した
- [ ] コンテキスト情報を構造化した
- [ ] コンテキスト情報をプロンプトに組み込んだ

#### 3.2 既存コードのコンテキスト活用（10点）
- [ ] 既存のTerraformコードを読み込む機能を実装した
- [ ] 既存コードの情報をコンテキストとして活用した
- [ ] コンテキストサイズを最適化した

### 4. AI Agent実践（20点）

#### 4.1 シンプルなAIエージェントの実装（15点）
- [ ] コード生成機能を実装した
- [ ] コード検証機能を実装した
- [ ] エラーハンドリングを実装した
- [ ] エージェントが正常に動作する

#### 4.2 エージェントアーキテクチャの理解（5点）
- [ ] エージェントの基本的なアーキテクチャを理解した
- [ ] LLM APIとの連携方法を理解した

### 5. AI x IaCを使ったEC2の設計・構築・検証（10点）

#### 5.1 EC2インスタンスの設計（3点）
- [ ] 自然言語指示でEC2インスタンスを設計した
- [ ] 要件を明確に定義した

#### 5.2 Terraformコード生成（3点）
- [ ] AIを活用してTerraformコードを生成した
- [ ] 生成されたコードが正しい形式である

#### 5.3 構築と検証（4点）
- [ ] EC2インスタンスを構築した
- [ ] 構築結果を検証した
- [ ] AWSコンソールまたはCLIで確認した

## 評価基準

### 優秀（90点以上）
- すべての項目を完了し、追加の改善や最適化を行った
- エージェントが安定して動作し、エラーハンドリングが適切
- コードの品質が高く、再利用可能な設計になっている

### 良好（70-89点）
- 必須項目をすべて完了した
- エージェントが基本的に動作する
- コードが正しく動作する

### 要改善（70点未満）
- 一部の項目が未完了
- エージェントが正常に動作しない
- コードにエラーがある
