# セッション1：環境セットアップとVPC/Subnet/EC2構築 評価チェックリスト

## 評価項目

### 1. VPC設計（15点）

#### 1.1 CIDRブロックの設計（5点）
- [ ] VPC CIDRブロックを適切に設計した（例: 10.0.0.0/16）
- [ ] サブネットCIDRブロックを適切に分割した
- [ ] CIDRブロックが重複していない

#### 1.2 可用性ゾーンの選択（5点）
- [ ] 複数の可用性ゾーンを選択した
- [ ] 可用性ゾーンの選択が適切である

#### 1.3 VPC設定（5点）
- [ ] DNSホスト名を有効化した
- [ ] DNSサポートを有効化した
- [ ] 適切なタグを設定した

### 2. サブネット設計（20点）

#### 2.1 パブリックサブネットの設計（10点）
- [ ] パブリックサブネットを2つ以上作成した
- [ ] 異なる可用性ゾーンに配置した
- [ ] パブリックIPの自動割り当てを有効化した
- [ ] 適切なCIDRブロックを割り当てた

#### 2.2 プライベートサブネットの設計（10点）
- [ ] プライベートサブネットを2つ以上作成した
- [ ] 異なる可用性ゾーンに配置した
- [ ] 適切なCIDRブロックを割り当てた

### 3. ルートテーブルとインターネットゲートウェイ（15点）

#### 3.1 インターネットゲートウェイ（5点）
- [ ] インターネットゲートウェイを作成した
- [ ] VPCにアタッチした

#### 3.2 ルートテーブル（10点）
- [ ] パブリックルートテーブルを作成した
- [ ] 0.0.0.0/0 → インターネットゲートウェイのルートを追加した
- [ ] パブリックサブネットとルートテーブルを関連付けた

### 4. EC2インスタンスの設計（20点）

#### 4.1 インスタンス設定（5点）
- [ ] 適切なインスタンスタイプを選択した（t3.micro推奨）
- [ ] 適切なAMIを選択した

#### 4.2 セキュリティグループの設計（10点）
- [ ] セキュリティグループを作成した
- [ ] SSH（ポート22）のインバウンドルールを設定した
- [ ] アウトバウンドルールを設定した
- [ ] 最小権限の原則に従った

#### 4.3 キーペアの設定（5点）
- [ ] キーペアを作成または既存のものを使用した
- [ ] EC2インスタンスにキーペアを設定した

### 5. Terraformコードの作成と実行（20点）

#### 5.1 Terraformコードの作成（10点）
- [ ] main.tfを作成した
- [ ] variables.tfを作成した
- [ ] outputs.tfを作成した
- [ ] terraform.tfvarsを作成した
- [ ] コードが正しい形式である

#### 5.2 Terraformの実行（10点）
- [ ] `terraform init`が成功した
- [ ] `terraform plan`で実行計画を確認した
- [ ] `terraform apply`でリソースを作成した
- [ ] エラーなく実行できた

### 6. 構築結果の検証（10点）

#### 6.1 AWSコンソールでの確認（5点）
- [ ] VPCが作成されていることを確認した
- [ ] サブネットが作成されていることを確認した
- [ ] EC2インスタンスが起動していることを確認した
- [ ] セキュリティグループが設定されていることを確認した

#### 6.2 AWS CLIでの確認（5点）
- [ ] AWS CLIでVPCを確認した
- [ ] AWS CLIでサブネットを確認した
- [ ] AWS CLIでEC2インスタンスを確認した
- [ ] 接続テストを行った（可能な場合）

- コードがベストプラクティスに従っている
- 適切なドキュメント化がされている

### 良好（70-89点）
- 必須項目をすべて完了した
- リソースが正常に構築された
- 基本的な検証が完了した

### 要改善（70点未満）

- 検証が不十分
