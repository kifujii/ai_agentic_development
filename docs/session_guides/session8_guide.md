## 📋 課題

### シナリオ

> **上司からの依頼**:
>
> 「今の EC2 は1台だけど、サーバーが落ちたらサービスが止まるのはまずい。
> 冗長構成にして、1台が落ちてもサービスが止まらないようにしてくれ。
> 動作確認で実際に1台落としても大丈夫なことを見せてほしい。」

### 機能要件

- EC2 が複数台（2台以上）で稼働し、それぞれ nginx が動作している
- ロードバランサー経由でアクセスでき、リクエストが複数台に分散される
- 1台の EC2 を停止しても、サービスが継続する

### 非機能要件

- Terraform でインフラを管理
- 各 EC2 のセットアップは user_data または Ansible で自動化

### ヒント

- AWS の ALB（Application Load Balancer）を使うのが定番です
- 「ALB を使った冗長構成を Terraform で作りたい」と Claude Code に相談してみましょう
- 動作確認として「1台を止めてからブラウザでアクセス」を試してみてください
