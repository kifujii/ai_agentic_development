## 📋 課題

### シナリオ

> **上司からの依頼**:
>
> 「来週の社内発表会で使うデモ用の Web アプリを用意してほしい。EC2 上で動くシンプルなものでいい。
> チーム内のメンバーがアクセスして使えるようにしておいてくれ。見た目もそこそこ良い感じにしてほしい。」

### 機能要件

- ブラウザからアクセスできる Web アプリケーションが EC2 上で動作する
- 何らかのインタラクティブな機能がある（フォーム入力、ボタン操作、データ表示など）
- チームメンバーが HTTP でアクセスできる

### 非機能要件

- Terraform で EC2 環境を管理（既存の環境を活用 or 新規構築）
- Ansible または user_data でアプリケーションのデプロイを自動化
- サーバー再起動後もアプリケーションが自動起動する

### ヒント

- アプリの内容は自由です（TODO リスト、チャット、ダッシュボード、クイズ、何でも OK）
- 言語・フレームワークも自由です（Claude Code に相談してください）
- 「どんなアプリを作ればいいかわからない」場合は、Claude Code に「デモ用の Web アプリのアイデアを5つ提案して」と聞いてみましょう

## 🛠️ 進め方のアドバイス

### 1. まず Plan モードで設計

```bash
claude --plan
```

課題の要件を伝えて、Claude Code とアーキテクチャを相談しましょう。

### 2. 小さく始めて段階的に構築

一度にすべてを作ろうとせず、**最小限の動くものを先に作り** → 少しずつ機能を追加していきましょう。

### 3. 詰まったら要件を整理

うまくいかないときは、**何ができていて何ができていないか** を Claude Code に伝えましょう。

### 4. 完璧でなくても OK

時間内に全ての要件を満たせなくても大丈夫です。**どこまでできたか、何を試したか** がこのセッションの成果です。
