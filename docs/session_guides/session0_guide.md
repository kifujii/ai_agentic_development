## Step 1: Claude Code を起動しよう — 🟢 お手本（5分）

> このStepでは **お手本** を用意しています。Claude Code との最初の出会いを楽しみましょう。

### やること

Claude Code を起動して、AI Agent がどう動くかを観察します。

### 手順

1. ターミナルで Claude Code を起動します：

```bash
claude
```

2. 以下のプロンプトを入力します：

```
このプロジェクトの構成を説明してください。どんなファイルやフォルダがあって、何のためのプロジェクトか教えてください。
```

3. **Claude Code が動く様子を観察しましょう**：
   - ファイルやフォルダを **自分で読みに行く** 動きをします
   - 読み取った内容をもとに回答を組み立てます

> 💡 **ここが ChatGPT との違い**: Claude Code は「ファイルを読む」「コマンドを実行する」といった **行動** を自律的に起こします。ただ質問に答えるだけではなく、必要な情報を自分で集めてから回答します。

### 覚えておくと便利なこと

| 操作 | 方法 |
|------|------|
| Claude Code を起動 | `claude` |
| 初期プロンプト付きで起動 | `claude "指示内容"` |
| 前回のセッションを続行 | `claude -c` |
| ヘルプを見る | 対話中に `/help` と入力 |
| セッションを終了 | `Ctrl+C` または `/exit` |

### スラッシュコマンドを使ってみよう

Claude Code の対話中に使える便利なコマンドがあります。いくつか試してみましょう：

| コマンド | 説明 |
|---------|------|
| `/help` | 使えるコマンド一覧を表示 |
| `/exit` | セッションを終了して bash に戻る |
| `/compact` | 会話の要約を作成してコンテキストを圧縮する |

> 💡 `/exit` で bash に戻った後、`claude -c` で同じ会話を再開できます。ワークショップ中、手動でコマンドを実行したい場面で活用してください。

## Step 2: Claude と壁打ちして自己紹介を作ろう — 🟢 お手本（5分）

> このStepでは Claude Code との **対話的なやりとり** と **承認ワークフロー** を体験します。AI Agent に一方的に指示するだけでなく、壁打ち相手として活用する感覚をつかみましょう。

### やること

Claude Code と対話しながら自分の自己紹介を作り、ファイル作成の承認フローを体験します。

### 手順

1. 以下のプロンプトを入力して、Claude と自己紹介の壁打ちを始めます：

```
ワークショップの自己紹介を一緒に考えてください。
まず、良い自己紹介を作るためにいくつか質問してください。
私の回答をもとに、practice/self_intro.md として自己紹介を作成してください。
```

> 💡 **複数行のプロンプトを入力するには `Alt + Enter` で改行します。** `Enter` だけだと送信されてしまうので注意してください。

2. **Claude Code との壁打ちを体験しましょう**：
   - Claude Code がいくつか質問してきます（名前、役割、経験など）
   - 質問に答えると、回答をもとに自己紹介を組み立ててくれます

3. **ファイル作成の承認フロー** を体験しましょう：
   - Claude Code が作成するファイルの内容（差分）を表示します
   - **内容を確認して承認** してください

4. **(任意) フィードバックを伝えて修正してもらいましょう**：

```
もう少しカジュアルなトーンにしてください。
```

   - 変更案が表示されたら、**一度却下** してみてください
   - 壊れたりしません。「やっぱり別の案で」と伝えれば、別の案を出してくれます

### このStepのポイント

| 体験 | Session 1以降では |
|------|-----------------|
| **AI Agent と壁打ちしながら成果物を作る** | 要件が曖昧なときに対話で詰めてからコードを生成 |
| **ファイル作成・編集の承認** | Terraform コードの生成時 |
| **コマンド実行の承認** | `terraform apply`, `ssh` 接続時 |
| **フィードバック → 修正のサイクル** | 生成されたコードをレビューして改善を繰り返す |

> 💡 **承認 = あなたに制御権がある証拠**です。Claude Code は勝手に実行しません。内容を確認してから進めるので、安心して使えます。

## Step 3: 自由に触ってみよう — 🟡 フリータイム（5分）

> このStepは **自由探索** です。何を試しても OK。ここでの経験が Session 1 以降の「プロンプトの書き方」の基礎になります。

### やること

Claude Code に色々な指示を出して、「どう伝えれば意図通りに動くか」を試行錯誤してみましょう。

### 試してみるアイデア

何をやるか迷ったら、以下を参考にしてください。どれから始めても OK です。

**🔰 まずはこれ: 曖昧 vs 具体的**

同じ目的で、曖昧な指示と具体的な指示を比べてみましょう：

```
practice/ にシェルスクリプトを作って
```

↓ 次にこちらを試す

```
practice/ フォルダに system_info.sh というシェルスクリプトを作成してください。

- OS情報（uname -a の結果）
- ディスク使用量（df -h の結果）
- 現在のユーザー名（whoami の結果）
を見やすく表示する内容にしてください。
実行権限を付与して、実行結果も見せてください。
```

→ 二つの結果を見比べてみてください。プロンプトの具体性で結果がどう変わりますか？

**💻 やりたいことを自由に試す**

- 「簡単なクイズゲームのシェルスクリプトを作って」
- 「このプロジェクトの README.md を要約して」
- 「practice/ に今日の日付で日報テンプレートを作って」
- 「practice/ に FizzBuzz を好きな言語で書いて」
- その他、何でも試してみてください！

**🔧 エラーが起きたら**

エラーが起きても焦らなくて大丈夫です。**エラーメッセージをそのまま Claude Code に伝えてください**。AI Agent が自動で修正を提案してくれます。

```
エラーが出ました。修正してください。
```

> 🎓 **気づいてほしいこと**: エラーを AI Agent に伝える → AI Agent が修正する、という流れは Session 1 以降で Terraform や Ansible のエラーに遭遇したときもまったく同じです。

## Step 4: ワークショップ環境を確認しよう（3分）

> Session 1 からは AWS 環境を使います。Claude Code に確認してもらいましょう。

### 手順

以下のプロンプトを入力してください：

```
以下を確認して結果を教えてください。
- terraform, ansible, aws cli がインストールされているか（バージョンも表示）
- aws sts get-caller-identity を実行して AWS に接続できるか
- 環境変数 TF_VAR_prefix が設定されているか
```

すべて OK なら、ワークショップを進める準備が整っています ✅

<details>
<summary>❓ ツールが見つからない場合</summary>

ツールはハンズオン環境にプリインストール済みです。見つからない場合は講師に確認してください。

</details>

<details>
<summary>❓ AWS認証が通らない場合</summary>

AWS 認証情報は環境に事前設定済みです。問題がある場合は講師に確認してください。

</details>

## 📝 振り返り（2分）

### Claude Code で体験したこと

| 体験 | Session 1以降でどう活きるか |
|------|--------------------------|
| **AI Agent がファイルを自動探索する** | 既存の Terraform コードを読んで追加・修正してくれる |
| **承認してから実行される** | `terraform apply` の実行を安心して判断できる |
| **具体的な指示ほど意図通りに動く** | 要件を明確に伝えて意図通りのインフラを構築できる |
| **エラーを伝えれば AI Agent が修正する** | Terraform / Ansible のエラーも同じ対処法で解決 |
| **複数の作業をまとめて依頼できる** | コード生成 → init → apply を一度の指示で自動化 |

### Session 1 に向けて

Session 1 では、ここで身につけた Claude Code の操作スキルを使って **実際の AWS インフラを構築** します。
プロンプトの書き方は Session 1 の Step 1 でお手本を見せますので、構えなくて大丈夫です。
