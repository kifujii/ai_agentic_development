## Step 1: Claude Code を起動しよう — 🟢 お手本（10分）

> このStepでは **お手本** を用意しています。Claude Code との最初の出会いを楽しみましょう。

### やること

Claude Code を起動して、AIエージェントがどう動くかを観察します。

### 手順

1. ターミナルで Claude Code を起動します：

```bash
claude
```

2. 以下のプロンプトを入力します：

```
このプロジェクトの構成を説明してください。どんなファイルやフォルダがあって、何のためのプロジェクトか教えてください。
```

3. **Claude Code が動く様子を観察しましょう**：
   - ファイルやフォルダを **自分で読みに行く** 動きをします
   - 読み取った内容をもとに回答を組み立てます

> 💡 **ここが ChatGPT との違い**: Claude Code は「ファイルを読む」「コマンドを実行する」といった **行動** を自律的に起こします。ただ質問に答えるだけではなく、必要な情報を自分で集めてから回答します。

### 覚えておくと便利なこと

| 操作 | 方法 |
|------|------|
| Claude Code を起動 | `claude` |
| 初期プロンプト付きで起動 | `claude "指示内容"` |
| 前回のセッションを続行 | `claude -c` |
| ヘルプを見る | 対話中に `/help` と入力 |
| セッションを終了 | `Ctrl+C` または `/exit` |

## Step 2: ファイル操作を体験しよう — 🟢 お手本（10分）

> このStepでは Claude Code の **承認ワークフロー** を体験します。Session 1 で `terraform apply` を承認するときの予行演習です。

### やること

Claude Code にファイル作成とコマンド実行を依頼して、承認の流れを体験します。

### 手順

1. 以下のプロンプトを入力します：

```
practice/ フォルダに self_intro.md というMarkdownファイルを作ってください。
内容は架空の自己紹介で、名前・好きな技術・一言メッセージを含めてください。
```

2. **ファイル作成の承認フロー** を体験しましょう：
   - Claude Code が作成するファイルの内容（差分）を表示します
   - **内容を確認して承認** してください

3. 続けて、以下を入力します：

```
今作ったファイルの内容をターミナルに表示してください。
```

4. **コマンド実行の承認フロー** を体験しましょう：
   - 実行しようとするコマンド（`cat practice/self_intro.md` など）が表示されます
   - **コマンドを確認して承認** してください

5. **(任意) 却下も試してみましょう**：

```
self_intro.md の内容を全く違う内容に書き換えてください。
```

   - 変更案が表示されたら、**一度却下** してみてください
   - 壊れたりしません。「やっぱり別の案で」と伝えれば、別の案を出してくれます

### このStepのポイント

| 承認の種類 | 表示される内容 | Session 1以降では |
|-----------|-------------|-----------------|
| **ファイル作成・編集** | ファイルの差分（追加/削除） | Terraform コードの生成時 |
| **コマンド実行** | 実行するコマンド | `terraform apply`, `ssh` 接続時 |

> 💡 **承認 = あなたに制御権がある証拠**です。Claude Code は勝手に実行しません。内容を確認してから進めるので、安心して使えます。

## Step 3: 自由に触ってみよう — 🟡 フリータイム（15分）

> このStepは **自由探索** です。何を試しても OK。ここでの経験が Session 1 以降の「プロンプトの書き方」の基礎になります。

### やること

Claude Code に色々な指示を出して、「どう伝えれば意図通りに動くか」を試行錯誤してみましょう。

### 試してみるアイデア

何をやるか迷ったら、以下を参考にしてください。どれから始めても OK です。

**🔰 まずはこれ: 曖昧 vs 具体的**

同じ目的で、曖昧な指示と具体的な指示を比べてみましょう：

```
practice/ にシェルスクリプトを作って
```

↓ 次にこちらを試す

```
practice/ フォルダに system_info.sh というシェルスクリプトを作成してください。

- OS情報（uname -a の結果）
- ディスク使用量（df -h の結果）
- 現在のユーザー名（whoami の結果）
を見やすく表示する内容にしてください。
実行権限を付与して、実行結果も見せてください。
```

→ 二つの結果を見比べてみてください。プロンプトの具体性で結果がどう変わりますか？

**💻 やりたいことを自由に試す**

- 「簡単なクイズゲームのシェルスクリプトを作って」
- 「このプロジェクトの README.md を要約して」
- 「practice/ に今日の日付で日報テンプレートを作って」
- 「practice/ に FizzBuzz を好きな言語で書いて」
- その他、何でも試してみてください！

**🔧 エラーが起きたら**

エラーが起きても焦らなくて大丈夫です。**エラーメッセージをそのまま Claude Code に伝えてください**。Agent が自動で修正を提案してくれます。

```
エラーが出ました。修正してください。
```

> 🎓 **気づいてほしいこと**: エラーを Agent に伝える → Agent が修正する、という流れは Session 1 以降で Terraform や Ansible のエラーに遭遇したときもまったく同じです。

## Step 4: ワークショップ環境を確認しよう（5分）

> Session 1 からは AWS 環境を使います。必要なツールが揃っているか、Claude Code に確認してもらいましょう。

### 手順

以下のプロンプトを入力してください：

```
以下のツールがインストールされているか確認して、バージョンとともに一覧表示してください。
- terraform
- ansible
- aws cli

また、aws sts get-caller-identity を実行して、AWS認証が通るか確認してください。
```

すべてのツールがインストール済みで、AWS認証が通れば OK ✅

> 💡 **ポイント**: 「複数のツールをまとめて確認して」という指示で、Agent が複数のコマンドを順次実行してくれます。Session 1 以降で「Terraform コードを作成して apply まで実行して」と指示するのと同じパターンです。

<details>
<summary>❓ ツールが見つからない場合</summary>

セットアップスクリプトを再実行してください：

```bash
./scripts/setup.sh
```

</details>

<details>
<summary>❓ AWS認証が通らない場合</summary>

- `.env` ファイルに AWS 認証情報が正しく設定されているか確認してください
- 詳しくは[環境セットアップガイド](../setup/ENVIRONMENT_SETUP.md)を参照

</details>

## 📝 振り返り（5分）

### Claude Code で体験したこと

| 体験 | Session 1以降でどう活きるか |
|------|--------------------------|
| **Agent がファイルを自動探索する** | 既存の Terraform コードを読んで追加・修正してくれる |
| **承認してから実行される** | `terraform apply` の実行を安心して判断できる |
| **具体的な指示ほど意図通りに動く** | 要件を明確に伝えて意図通りのインフラを構築できる |
| **エラーを伝えれば Agent が修正する** | Terraform / Ansible のエラーも同じ対処法で解決 |
| **複数の作業をまとめて依頼できる** | コード生成 → init → apply を一度の指示で自動化 |

### Session 1 に向けて

Session 1 では、ここで身につけた Claude Code の操作スキルを使って **実際の AWS インフラを構築** します。
プロンプトの書き方は Session 1 の Step 1 でお手本を見せますので、構えなくて大丈夫です。

> 💡 Session 1 に進む前に、SSH鍵ペアの生成を忘れずに行ってください（[セッション1ガイド](session1_guide.md) の「事前準備」参照）。
