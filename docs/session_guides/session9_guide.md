## 📋 課題

### シナリオ

> **上司からの依頼**:
>
> 「サーバーの状態を定期的にチェックして、異常があったら自動で通知される仕組みを作ってほしい。
> CPU やメモリが一定以上になったら通知が飛ぶようにして、
> できれば定期的なヘルスチェックレポートも自動生成してほしい。」

### 機能要件

- CPU やメモリの使用率が閾値を超えたときにアラームが発火する
- 何らかの通知手段（SNS、メール、Slack 等）で通知される
- 定期的にサーバー状態をチェックし、レポートを生成する仕組みがある

### 非機能要件

- CloudWatch Agent でメトリクスを収集（セッション5の環境を活用）
- AWS CLI または Terraform でアラーム・通知を構成
- Ansible でヘルスチェックを自動化

### ヒント

- セッション5で作った CloudWatch 環境を拡張しましょう
- SNS（Simple Notification Service）で通知先（メールアドレス等）を設定できます
- 「CloudWatch Alarm + SNS で通知を設定したい」と Claude Code に相談してみましょう
