# セッション10：ゼロからシステム構築チャレンジ（1.5時間・任意）

> このセッションは **任意（発展課題）** です。時間に余裕がある方向けです。

## 学習目標

- ゼロから設計・実装・動作確認までを完遂する実践力を身につける
- これまでのセッションで学んだスキルをフル活用して完全に身につける。

## シナリオ

社内ハッカソンが開催されます。テーマは **「1.5時間で動くものを作れ」** です。

以下の3つのお題から1つを選んでください。

---

## お題 A：社内ステータスページ

> 「うちのサービス、今動いてるの？」という質問に毎回答えるのが面倒です。
> **誰でも見られるステータスページ**を作ってください。

- 監視対象のサーバーの状態を自動取得して表示する Web ページ
- 正常 / 異常が一目でわかる
- 手動で更新する必要がない

---

## お題 B：デプロイパイプライン

> 開発チームから「サーバーにコードを上げるのに毎回 SSH してるんだけど...」と言われました。
> **コマンド一発でデプロイできる仕組み**を作ってください。

- ローカルのコード変更をサーバーに反映するワンコマンドの仕組み
- 失敗したら前のバージョンに戻せる
- デプロイ履歴が残る

---

## お題 C：複数環境の管理

> 「開発環境とステージング環境がほしい。本番と同じ構成で。」と言われました。
> **同じコードから複数の環境を作れる仕組み**を構築してください。

- 1つの Terraform コードベースから dev / staging / prod の3環境を作成
- 環境ごとにパラメータ（インスタンスサイズ等）を変えられる
- 1つの環境だけを壊して作り直せる

---

## 🎯 ゴール

以下の状態になっていれば完了です：

1. **選んだお題に対して動くものが存在する**
   - 完璧でなくて良い。**デモできるレベル**で動いていれば OK
2. **構成を説明できる**
   - 何を作ったか、なぜその構成にしたかを口頭で説明できる
   - AI に設計ドキュメントを作ってもらっても OK

---

## 進め方

時間配分は自由ですが、参考として：

| フェーズ | 目安 | やること |
|---------|------|---------|
| 設計 | 20分 | Plan モード（対話中に `Shift + Tab` または `/plan` で切り替え）で AI と構成を議論 |
| 構築 | 50分 | Terraform / Ansible で実装 |
| 仕上げ | 20分 | 動作確認、ドキュメント作成 |

### 重要なポイント

- **完璧を目指さない**。動くものを最短で作ることを優先してください
- 途中で詰まったら、アプローチを変える判断も大事です
- AI に「今の状況を整理して。次に何をすべき？」と聞くのは有効な戦略です

## 📚 事前準備

> ⚠️ **環境変数が未設定の場合**: `echo $TF_VAR_prefix` で値が表示されない場合は講師に確認してください。

- これまでのセッションで構築した環境を再利用しても、新しく作り直しても構いません
- お題ごとの推奨前提:
  - **お題 A**: Session 4-5 の Ansible 環境があると進めやすい
  - **お題 B**: Session 4 の Ansible 基本操作に慣れていれば OK
  - **お題 C**: Session 1-2 の Terraform 基本操作に慣れていれば OK（workspace / tfvars は AI と一緒に学びます）

---

## ⚠️ リソースの削除

このセッションで作成したリソースは、完了後にすべて削除してください。

```bash
# Terraform で作成したリソース
terraform -chdir=terraform/<作成したディレクトリ> destroy
```

> 💡 お題によって削除対象が異なります。AWS CLI で作成したリソース（S3 バケット、IAM ロールなど）がある場合は、Claude Code に「このセッションで作成した AWS リソースをすべて削除して」と依頼するのも有効です。
