# セッション3：EC2 で動的 Web アプリを作ろう（任意・45分）

> このセッションは **任意（発展課題）** です。セッション2が完了し、余裕がある方向けです。

## 🎯 このセッションのゴール

セッション2では静的HTMLをnginxで公開しました。このセッションでは **Agent に一括指示** して、**データベース連携の動的 Web アプリ**（ゲストブック）を作成し、ブラウザで動作確認します。

![目標構成](../images/session3_target.svg)

> 🎓 **セッション2からのステップアップ**: 静的HTML → DB連携の動的Webアプリへ

> 💡 **ポイント**: このセッションでは**手動でSSHやコマンド実行はしません**。すべてAgentに指示します。

## Step 1: Agent にアプリの作成・デプロイを一括指示しよう（30分）

### やること

ContinueのAgentに、**PHP環境のセットアップからアプリ作成・デプロイまで一括で指示**します。

データベースには **SQLite** を使います。RDSなどの外部サービスは不要で、EC2内で完結します。

### ゴール

Agentに以下を一括で実行してもらう：

1. EC2にPHP環境をセットアップ（PHP + SQLite拡張のインストール、nginx設定）
2. ゲストブック（名前とメッセージの投稿・一覧表示）PHPアプリを作成
3. EC2にデプロイして動作する状態にする

<details>
<summary>📝 プロンプト例</summary>

```
以下の作業をすべて実行してください。

■ 前提情報
- EC2のIP: <IPアドレス>
- SSH鍵: ~/.ssh/training-key
- SSHユーザー: ec2-user
- nginxはEC2上で起動済み（セッション2で構築済み）

■ やること（SSHでEC2に接続して実行してください）
1. PHP環境セットアップ
   - php-fpm, php-pdo, php-sqlite3（php-pdo_sqlite相当） をインストール
   - php-fpm を起動・有効化
   - nginx に PHP 用の server ブロックを追加して再起動
     （ポート80で PHP を処理できるように設定）

2. ゲストブックアプリ作成
   - /usr/share/nginx/html/index.php を作成
   - SQLite データベース（/var/lib/nginx/guestbook.db）を使用
   - テーブル: messages (id, name, message, created_at)
   - 機能: メッセージ一覧表示（新しい順）+ 投稿フォーム
   - モダンで見やすいデザイン（CSSインライン）
   - データベースファイルの権限を nginx ユーザーが読み書きできるように設定

3. テストデータを1件投入して動作確認
```

</details>

### Agentの作業を観察

Agentが以下のような手順で作業を進める様子を観察しましょう：

- SSH接続 → パッケージインストール → 設定変更 → サービス再起動
- PHPファイル作成 → 配置 → 権限設定
- エラーが出た場合は自動で修正を試みる

---

## Step 2: ブラウザで動作確認しよう（10分）

ブラウザで `http://<EC2のIPアドレス>` にアクセスします。

- ✅ ゲストブックが表示される
- ✅ テストデータが一覧に表示されている
- ✅ フォームからメッセージを投稿できる
- ✅ 投稿後、一覧に新しいメッセージが表示される

上記がすべて確認できれば **セッション3完了** 🎉

<details>
<summary>❓ 表示されない場合</summary>

Agentに以下のように伝えてトラブルシューティングを依頼しましょう：

```
EC2（<IPアドレス>）にSSHで接続して、以下を確認・修正してください。
- nginx が起動しているか
- php-fpm が起動しているか
- /usr/share/nginx/html/index.php が存在するか
- nginx のエラーログに何か出ていないか
- SQLite データベースファイルの権限は正しいか
```

</details>
