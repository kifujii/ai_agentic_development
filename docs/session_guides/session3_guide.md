# セッション3：Webサーバーを冗長構成にしよう（任意・30分）

> このセッションは **任意（発展課題）** です。セッション2が完了し、余裕がある方向けです。

## シナリオ

あなたはチームリーダーから以下の依頼を受けました。

> 「いま Web サーバーが1台だけで動いてるんだけど、もう1台追加して冗長構成にしてほしい。片方のサーバーが落ちても、ユーザーがページを見られるようにしたい。Terraform で管理してね。」

---

## ゴール

- Web ページが表示される EC2 が 2台動いている
- 1台を停止しても、もう1台でページが表示できる
- Terraform のコードで台数が管理されている
- 確認が終わったら、コードとインフラを1台構成に戻す（後続セッションのため）

---

## 事前準備

> ⚠️ **環境変数が未設定の場合**: `echo $TF_VAR_prefix` で値が表示されない場合は講師に確認してください。

- **セッション2が完了**していること（user_data 付きの EC2 が起動し、nginx に HTTP でアクセスできる状態）

---

## 進め方

このセッションには手順やプロンプト例はありません。**Plan モードで AI と相談しながら、自分で進め方を考えてください。**

1. Claude Code を **Plan モード**（`claude --plan` で起動、または対話中に `Shift + Tab` で切り替え）にして、「何が必要か」を相談する
2. 方針が決まったら通常モードに戻して実装する
3. 2台ともブラウザで動作確認する
4. 1台を止めてもページが見られることを確認する
5. 最後に1台構成に戻す

<details>
<summary>ヒント（まず自分で考えてから開いてください）</summary>

- Terraform には同じリソースを複数作る仕組みがあります
- 「冗長化」にもいろいろなやり方があります。ロードバランサー（ALB）を使う方法、単にサーバーを増やす方法... どれが今回のケースに合うか、AI に相談してみましょう
- 既存の EC2 が壊れないように変更するにはどうすればいいか、も考えるポイントです
- コードとインフラの状態が一致しなくなったとき、何が起きるかも体験してみてください

</details>
