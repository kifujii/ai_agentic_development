# 生成AI活用トレーニングメニュー詳細ドキュメント

## トレーニング概要

### 基本情報
- **期間**: 2日間（合計8時間、1日4時間）
- **形式**: ハンズオン形式のバイブコーディング
- **環境**: OpenShift DevSpaces + AWS
- **技術スタック**: Terraform, Ansible, 生成AIエージェント開発
- **前提条件**: 事前勉強会で生成AIとIaC（Terraform/Ansible）の基礎を学習済み

### 学習目標
- Prompt Engineering、Context Engineering、AI Agentの実践的スキルを習得
- Terraformを使ったAWSインフラの自動構築を実践
- Ansibleを使ったシステム運用の自動化を実践
- 生成AIエージェントによるIaCコード生成・実行の自動化を実装
- 統合的なインフラ管理タスクの自動化エージェントを開発

---

## 1日目：AI x IaC基礎実践とシステム構築（4時間）

### セッション0：AI x IaC基礎実践（必須）（1.5時間）

#### 目標
Prompt Engineering、Context Engineering、AI Agentの実践を通じて、AI x IaCの基礎を習得する。

#### 学習内容

**1. 環境セットアップ（15分）**
- OpenShift DevSpaces環境の確認と操作
- AWS CLI/認証情報の設定（Admin権限のアクセスキー・シークレットキーを使用）
- 生成AI APIキーの設定確認
- 必要なツールのインストール確認（Terraform, Ansible, Python/Node.js）

**2. Prompt Engineering実践（30分）**
- Terraformコード生成のための効果的なプロンプト設計
  - 明確な要件定義の重要性
  - リソース仕様の具体的な記述方法
  - 段階的なプロンプト最適化の実践
- プロンプトテンプレートの作成と改善
  - 再利用可能なプロンプトテンプレートの設計
  - コンテキスト情報の組み込み方法
- 生成コードの品質向上テクニック
  - 複数回生成による品質比較
  - フィードバックループの構築

**3. Context Engineering実践（20分）**
- コンテキスト情報の構造化と管理
  - AWSリソース情報のコンテキスト化
  - Terraformモジュールや既存コードのコンテキスト活用
  - コンテキストの効率的な提供方法
  - コンテキストサイズの最適化

**4. AI Agent実践（25分）**
- シンプルなAIエージェントの実装
  - コード生成→検証→実行の流れ
  - エージェントの基本的なアーキテクチャ理解
  - エージェントとLLM APIの連携方法
  - エラーハンドリングの基本

**5. AI x IaCを使ったEC2の設計・構築・検証（実践）（20分）**
- 自然言語指示によるEC2インスタンスの設計
- AIを活用したTerraformコード生成
- 生成コードの検証と修正
- EC2インスタンスの構築と動作確認

#### 成果物
- Prompt Engineering、Context Engineering、AI Agentの実践成果
- AIを活用して構築されたEC2インスタンス
- プロンプトテンプレート集
- シンプルなAIエージェントの実装コード

#### 参考資料
- `docs/session_guides/session0_guide.md`
- `templates/ai_agents/simple_agent_template.py`

### セッション1：環境セットアップとVPC/Subnet/EC2構築（必須）（1時間）

#### 目標
AWSインフラの基本構成を手動で構築し、理解を深める。

#### 学習内容

**1. VPC設計（15分）**
- CIDRブロックの設計（10.0.0.0/16など）
- 可用性ゾーンの選択
- VPCの基本概念とベストプラクティス

**2. サブネット設計（15分）**
- パブリック/プライベートサブネットの設計
- CIDRブロックの分割方法
- ルートテーブルの理解

**3. EC2インスタンスの設計（15分）**
- インスタンスタイプの選択
- セキュリティグループの設計（最小権限の原則）
- キーペアの作成と管理
- IAMロールの設定（必要に応じて）

**4. Terraformコードの手動作成と実行（15分）**
- `terraform init` - プロバイダーの初期化
- `terraform plan` - 実行計画の確認
- `terraform apply` - リソースの作成
- `terraform destroy` - リソースの削除（必要に応じて）

**5. 構築結果の検証（10分）**
- AWSコンソールでの確認
- AWS CLIでの確認
- 接続テスト（SSHなど）

#### 成果物
- VPC、Subnet、EC2インスタンスが構築された状態
- Terraformコード（手動作成）
- 構築結果の検証レポート

#### 参考資料
- `docs/session_guides/session1_guide.md`
- `sample_code/terraform/vpc_subnet_ec2/`

## トレーニング設計のポイント

### 生成AIエージェント開発の3要素

1. **Terraformコード生成・実行の自動化**（セッション2）
   - セッション0で学んだPrompt Engineering、Context Engineering、AI Agentの基礎を活用
   - 自然言語からTerraformコードへの変換
   - コード生成、検証、自動実行の一連の流れ

2. **Ansible Playbook生成・実行の自動化**（セッション4）
   - 自然言語からAnsible Playbookへの変換
   - Playbook生成、検証、自動実行の一連の流れ

3. **インフラ管理タスクの自動化**（セッション5）
   - Terraform/Ansibleを統合した包括的なインフラ管理
   - 複雑なワークフローの自動化

### 時間配分の最適化

- **必須項目**: 約7時間
  - **1日目（4時間）**
    - セッション0: AI x IaC基礎実践（1.5時間）
    - セッション1: 環境セットアップとVPC/Subnet/EC2構築（1時間）
    - セッション2: Terraform自動化エージェント開発（1.5時間）
  - **2日目（4時間）**
    - セッション3: Ansible運用基礎（1時間）
    - セッション4: Ansible自動化エージェント開発（1.5時間）
    - セッション5: インフラ管理タスク自動化エージェント開発（1.5時間）
- **任意項目**: 約1時間（セッション6: Webシステム構築）
- **余裕時間**: 各セッションに5-10分のバッファを確保

## 成果物と評価

### 各セッションの成果物

- **セッション0**: Prompt Engineering、Context Engineering、AI Agentの実践成果、AIを活用して構築されたEC2インスタンス
- **セッション1**: VPC、Subnet、EC2インスタンスが構築されたAWS環境
- **セッション2**: Terraformコード生成・実行を自動化する生成AIエージェント
- **セッション3**: サーバー再起動を自動化するAnsible Playbook
- **セッション4**: Ansible Playbook生成・実行を自動化する生成AIエージェント
- **セッション5**: インフラ管理タスクを統合的に自動化する生成AIエージェント
- **セッション6（任意）**: Webアプリケーションが動作するインフラ環境

### 最終成果物

- **インフラ環境**: AWS上に構築されたVPC/Subnet/EC2環境（Webシステム含む）
- **生成AIエージェント**: 3つのエージェント（Terraform、Ansible、統合管理）

- **実装コードのレビュー**: エージェントの設計と実装品質

- **実用性**: 実際の業務で使えるレベルの自動化が実現できているか

詳細な評価チェックリストは `evaluation/` ディレクトリを参照してください。
