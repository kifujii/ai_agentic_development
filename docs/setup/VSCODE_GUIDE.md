# ブラウザ版 VSCode（code-server）操作ガイド

> このハンズオンでは、ブラウザ上で動作する VSCode（code-server）を使います。通常のデスクトップ版 VSCode とほぼ同じ操作感で使えます。

## 画面構成

```
┌──────────────────────────────────────────────────┐
│  メニューバー（ファイル / 編集 / 表示 ...）          │
├────────┬─────────────────────────────────────────┤
│        │                                         │
│ サイド  │           エディタ領域                    │
│ バー    │     （ファイルの編集はここで行う）          │
│        │                                         │
│ ファイル ├─────────────────────────────────────────┤
│ 一覧    │                                         │
│ など    │           ターミナル領域                   │
│        │   （Claude Code やコマンドはここで実行）     │
│        │                                         │
└────────┴─────────────────────────────────────────┘
```

- **サイドバー（左）**: ファイル一覧、検索、Git 操作などにアクセス
- **エディタ領域（右上）**: ファイルの内容を表示・編集
- **ターミナル領域（右下）**: コマンドや Claude Code を実行

---

## ターミナルの操作

このハンズオンではターミナルを頻繁に使います。

### ターミナルを開く

| 操作 | 方法 |
|-----|------|
| メニューから | 「ターミナル」→「新しいターミナル」 |
| ショートカット | `` Ctrl + Shift + ` ``（バッククォート） |
| ショートカット | `Ctrl + Shift + C` |

### ターミナルを複数使う

Claude Code を実行しながら別のコマンドも打ちたい場面があります。ターミナルは複数開けます。

- ターミナル領域の右上にある **＋** ボタンで新しいターミナルを追加
- ターミナル領域の右上のドロップダウンで、ターミナルを切り替え

> **Tip**: Claude Code 用のターミナルと、自分でコマンドを実行するターミナルを分けておくと便利です。

### ターミナル領域の表示サイズを変える

ターミナル領域とエディタ領域の境界線をドラッグすると、ターミナルの表示領域を広げたり狭めたりできます。Claude Code の出力をじっくり見たいときは広げましょう。

## ファイル操作

### ファイルを開く

| 操作 | 方法 |
|-----|------|
| サイドバーから | ファイルエクスプローラーでファイルをクリック |
| クイックオープン | `Ctrl + P` → ファイル名を入力 |

`Ctrl + P` は特に便利です。ファイル名の一部を入力するだけで候補が表示されます。

```
例: 「main」と入力 → terraform/vpc-ec2/main.tf が候補に出る
```

### ファイルを保存する

| 操作 | 方法 |
|-----|------|
| 上書き保存 | `Ctrl + S` |
| 全ファイル保存 | `Ctrl + K` → `S` |

> **注意**: Claude Code がファイルを編集した場合は自動的に保存されるため、手動で保存する必要はありません。自分でエディタから直接編集した場合のみ `Ctrl + S` で保存してください。

## よく使うショートカット

| ショートカット | 機能 |
|--------------|------|
| `Ctrl + P` | クイックオープン（ファイル名で検索して開く） |
| `Ctrl + Shift + F` | ワークスペース全体を検索 |
| `Ctrl + S` | ファイルを保存 |
| `` Ctrl + Shift + ` `` | 新しいターミナルを開く |
| `Ctrl + B` | サイドバーの表示/非表示 |
| `Ctrl + Z` | 元に戻す |
| `Ctrl + Shift + Z` | やり直す |
| `Ctrl + /` | 選択行をコメントアウト/解除 |
| `Ctrl + D` | 同じ単語を次の出現箇所まで選択 |

## ブラウザ版特有の注意点

### ショートカットの競合

ブラウザ自体のショートカットと VSCode のショートカットが競合する場合があります。

| 操作 | ブラウザの動作 | 対処法 |
|-----|-------------|-------|
| `Ctrl + W` | ブラウザのタブを閉じる | VSCode のタブを閉じたいときは、タブの × ボタンをクリック |
| `Ctrl + N` | ブラウザの新しいウィンドウ | VSCode で新規ファイルを作りたいときは、メニュー → ファイル → 新規ファイル |
| `Ctrl + T` | ブラウザの新しいタブ | — |

### コピー＆ペースト

ターミナルへのコピー＆ペーストは通常どおり `Ctrl + C` / `Ctrl + V` で動作します。ただし、ターミナル内で `Ctrl + C` は実行中のプロセスの中断としても使われます。

- **テキストを選択している状態**: `Ctrl + C` = コピー
- **テキストを選択していない状態**: `Ctrl + C` = プロセスの中断

### ページを閉じる前に

ブラウザのタブを閉じても作業内容は EC2 上に保存されています。次回アクセスすれば続きから作業できます。ただし、ターミナルで実行中だった Claude Code のセッションは終了するため、再開するには `claude -c` を使ってください。

## 困ったときは

| 困りごと | 対処法 |
|---------|-------|
| ターミナルが見当たらない | `` Ctrl + Shift + ` `` で新しいターミナルを開く |
| 画面がごちゃごちゃした | `Ctrl + B` でサイドバーを閉じてスペースを確保 |
| ファイルがどこにあるかわからない | `Ctrl + P` でファイル名を検索 |
| 文字が小さい／大きい | `Ctrl + +` / `Ctrl + -` で拡大・縮小 |
| 画面が固まった | ブラウザをリロード（`F5`）すれば復帰する |
