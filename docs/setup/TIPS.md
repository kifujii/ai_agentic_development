# Tips: VSCode & Claude Code の使い方

ワークショップ中に役立つ操作方法をまとめています。困ったときに参照してください。

---

## 🖥️ VSCode ショートカット

| 操作 | Windows / Linux | Mac |
|------|----------------|-----|
| **ターミナルを開く（新規）** | `Ctrl+Shift+C` | `Cmd+Shift+C` |
| **ターミナルパネルの表示/非表示** | `` Ctrl+` `` | `` Cmd+` `` |
| **ファイルエクスプローラーの表示/非表示** | `Ctrl+B` | `Cmd+B` |
| **ファイルを素早く開く** | `Ctrl+P` | `Cmd+P` |
| **ファイル内容を検索** | `Ctrl+F` | `Cmd+F` |
| **全ファイルから検索** | `Ctrl+Shift+F` | `Cmd+Shift+F` |
| **エディタを左右に分割** | `Ctrl+\` | `Cmd+\` |
| **ターミナルを左右に分割** | ターミナル右上の分割アイコン | 同左 |

> 💡 **おすすめの画面構成**: エディタ（上部）+ ターミナル2つ（下部に分割）。1つは Claude Code 用、もう1つは手動コマンド用です。

## 🤖 Claude Code の操作

### 起動と終了

| 操作 | コマンド | 説明 |
|------|---------|------|
| 起動 | `claude` | 新しいセッションを開始 |
| プロンプト付きで起動 | `claude "指示内容"` | 起動と同時に指示を送る |
| セッションを再開 | `claude -c` | 前回のセッションを続行 |
| 終了 | `/exit` または `Ctrl+C` | bash に戻る |

### スラッシュコマンド（対話中に使用）

| コマンド | 説明 |
|---------|------|
| `/help` | 使えるコマンド一覧を表示 |
| `/exit` | セッションを終了して bash に戻る |
| `/compact` | 会話の要約を作成してコンテキストを圧縮する（長い会話で有用） |

### Plan モード

```bash
claude --plan
```

Plan モードでは Claude Code が**コードの変更を行わず**、構成や設計について対話的に相談できます。実装に入る前に「何を作るか」を整理するのに便利です。

## 🔄 bash と Claude Code の切り替え

ワークショップ中は **bash（通常のターミナル）** と **Claude Code** を行き来する場面があります。

### 方法1: ターミナル2つ使う（おすすめ）

1. ターミナルタブの「+」ボタンで **2つ目のターミナル** を追加
2. **ターミナル1**: Claude Code 用（`claude` を起動したまま）
3. **ターミナル2**: 手動コマンド用（`check.sh` の実行など）
4. タブをクリックして切り替え

### 方法2: 一時的に bash に戻る

1. Claude Code で `/exit` と入力 → bash に戻る
2. 手動コマンドを実行
3. `claude -c` で同じ会話を再開

> 💡 `check.sh` などのチェックスクリプトは **Claude Code の外（bash）** で実行してください。

## ⚠️ よくある操作ミス

| 状況 | 対処法 |
|------|--------|
| Claude Code が長時間応答しない | `Ctrl+C` で中断し、再度指示を出す |
| 間違って承認してしまった | 落ち着いて状況を Claude Code に伝える。`terraform destroy` で元に戻せることが多い |
| ターミナルが固まった | 新しいターミナルタブを開く（`Ctrl+Shift+C` / `Cmd+Shift+C`） |
| 環境変数が消えた | `echo $TF_VAR_prefix` で確認。表示されない場合は講師に確認 |
| Claude Code の会話が長くなって遅い | `/compact` で会話を圧縮、または `/exit` → `claude` で新しいセッション開始 |
